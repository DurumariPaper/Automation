### Context
n8n을 활용하여 디스코드(Discord) 메시지를 트리거로 받아, 이를 TIL(Today I Learned) 형식으로 변환한 뒤 GitHub 저장소에 자동으로 반영하는 워크플로우를 구축함. 전체 프로세스는 디스코드 메시지 수신, AI 에이전트를 통한 텍스트 구조화, GitHub 파일 Push, 그리고 README.md 인덱스 갱신 및 완료 알림 단계로 구성됨.

### Core
워크플로우는 각 노드 간의 데이터 흐름(Data Flow)을 이해하는 것이 핵심임. 주요 설정 단계는 다음과 같음.

* 트리거(Trigger): Discord 노드를 통해 특정 채널의 메시지를 감지하고 JSON 형태로 데이터를 추출함.
* 프로세싱(Processing): AI Agent 노드(LangChain 기반)를 사용하여 입력된 메시지를 TIL 마크다운 포맷으로 변환.
* 데이터 저장(Storage): GitHub 노드를 사용하여 파일 생성 및 커밋을 수행.
* 인덱싱(Indexing): Read/Write File 노드를 활용하여 기존 README.md에 새로운 TIL 링크를 추가.

```javascript
// AI Agent를 통한 데이터 구조화 로직 예시
const tilTemplate = `
# ${message.title}

### Context
${message.context}

### Core
${message.content}

### Insight
${message.insight}
`;
```

### Insight
n8n을 활용한 자동화의 핵심은 각 노드가 받아들이는 입력(Input) 데이터 구조와 출력(Output)으로 전달되는 스키마(Schema)를 명확히 파악하는 것임. 노드 간의 데이터 연결은 JSON 기반의 객체 전달이므로, 각 단계에서 데이터 필드명이 유지되는지 확인하는 것이 필수적임. 도구 활용 능력만큼이나 시스템의 데이터 흐름을 직관적으로 읽어내는 설계 역량이 자동화의 성패를 결정함.
